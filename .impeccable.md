# Google Docs Clone - Design Context

## Design Context

### Users
- **Target:** Knowledge workers, students, and teams who need to create, edit, and share documents
- **Context:** Desktop browser usage, creating documents for personal use or sharing with colleagues and clients
- **Job to be done:** Write and collaborate on documents efficiently with minimal friction

### Brand Personality
- **Voice:** Professional, clear, and approachable
- **Tone:** Business-appropriate without being cold or corporate
- **3-word personality:** Clean, Reliable, Focused
- **Emotional goals:** Confidence that work is safe, clarity in finding what you need, productivity through minimal friction

### Aesthetic Direction
- **Visual tone:** Google Docs/Notion-inspired light mode. Clean, minimal, generous whitespace. Professional without being sterile
- **References:** Google Docs, Notion, Linear — clean typography, ample white space, subtle interactions
- **Anti-references:** Cluttered toolbars, excessive color, busy gradients, dated enterprise software
- **Theme:** Light mode only — white/light gray backgrounds with dark text for maximum readability

### Design Principles
1. **Content First** — The document content is the hero. UI chrome stays minimal and out of the way during editing
2. **Predictable Patterns** — Users should never have to guess how something works. Standard patterns for buttons, forms, and navigation
3. **Progressive Disclosure** — Show only what's needed. Advanced options hidden until requested
4. **Accessible by Default** — WCAG AA contrast ratios, keyboard navigation, clear focus states
5. **Performance Perception** — Snappy interactions, loading states that communicate progress, no perceptible lag

### Typography
- **Font family:** `Inter, -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`
- **Scale:** 12 / 14 / 16 / 18 / 20 / 24 / 32 / 48px
- **Weights:** 400 (regular), 500 (medium), 600 (semibold), 700 (bold)
- **Line heights:** 1.25 (tight), 1.5 (normal), 1.75 (relaxed)

### Spacing
- **Base unit:** 4px
- **Scale:** 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64px
- **Screen padding:** 24px horizontal

### Shadows / Elevation
- Subtle shadows for light surfaces:
- `--shadow-sm`: `0 1px 2px rgba(0, 0, 0, 0.05)`
- `--shadow-md`: `0 4px 6px -1px rgba(0, 0, 0, 0.1)`
- `--shadow-lg`: `0 10px 15px -3px rgba(0, 0, 0, 0.1)`
- `--shadow-xl`: `0 20px 25px -5px rgba(0, 0, 0, 0.1)`

### Motion
- **Duration:** 150ms (micro), 250ms (standard), 350ms (emphasis)
- **Easing:** `ease-in-out` for state changes, `ease-out` for entrances
- **Principle:** Smooth and professional — never bouncy or playful

### Icon Style
- Filled icons, 24px default size
- Accent color for active/selected states
- Subtle hover states

### Global Usage of Common Styles
- CSS variables for colors, radius, shadows, and typography are defined in `src/styles/globals.css`
- Use common styling when available — do not hardcode values
- Follow the spacing scale strictly for consistency
