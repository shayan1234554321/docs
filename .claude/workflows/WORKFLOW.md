# Workflows

Start here:
1. **First run `teach`** when starting if you haven't used one
2. **Or** run each workflow individually:
   - [Teach context](teach-context) — Give your project context
   - [Teach PR review](teach-pr-review) — Create a PR review workflow for your tech stack
   - [Teach impeccable](teach-impeccable) — Create a new design language
   - [Extract UI](extract) — Create a design language from an existing design

---

## System Workflows

| Workflow | When to Use |
|----------|-------------|
| `teach` | Set up AI context and workflows for a new project |
| `teach-context` | Create CLAUDE.md context file for a project/directory |
| `teach-pr-review` | Prepare PR review workflow for your tech stack |
| `teach-impeccable` | Establish new design language from scratch |
| `extract` | Analyze and document existing design patterns |

---

## Review Workflows

| Workflow | When to Use |
|----------|-------------|
| `pr-review` | Review PR for bugs, code quality, security |

---

## Code Workflows

| Workflow | When to Use |
|----------|-------------|
| `optimize` | Check code quality, linting, formatting, best practices, file splitting if needed |
| `overdrive` | Large data optimization like millions of data, complex algorithms |
| `adapt-file` | Adapt a file to project layout, naming, folder structure as in CLAUDE.md |
| `audit` | Check accessibility, performance, and security |
| `change-layout` | Change how work is organized in the project, which effects CLAUDE.md file |

---

## Design Workflows

| Workflow | When to Use |
|----------|-------------|
| `adapt` | Make UI responsive to different screen sizes and devices |
| `animate` | Add animations to UI elements |
| `arrange` | Fix spacing or layout issues |
| `audit` | Check UI for accessibility issues and improvements |
| `bolder` | Add visual impact when design looks bland or generic |
| `clarify` | Improve unclear labels, error messages, or UX writing |
| `colorize` | Add warmth/vibrancy when design looks gray or dull |
| `critique` | Get feedback on UI design |
| `delight` | Add moments of joy, personality, unexpected touches |
| `normalize` | Make UI consistent with brand guidelines |
| `onboard` | First-time users, empty states, getting started flows |
